## Features

- **marker-delimited CLAUDE.md template region check (#1439)**
Replace the binary contiguous-substring CLAUDE.md compliance check
with a marker-delimited region check: exactly one
vergil:template:claude-md begin/end pair whose content must equal the
canonical consumer template, with first-divergent-line diagnostics
naming both the template line and the absolute CLAUDE.md line.
Repo-local content outside the markers never affects compliance.

Repos without markers fall back to the legacy contiguous-substring
check during the migration window. vrg-repo-init now scaffolds
CLAUDE.md with the markers in place.

- **per-profile nested virtualization via the [vm] cascade (#1449)**
* feat(vm-spec): per-profile nested virtualization via the [vm] cascade (#1447)

Parse 'nested = true|false' in the repo [vm]/[vm.<role>] cascade
(boolean scalar, last-wins like the footprint scalars, default false)
and drive both Lima halves together at create time:
.nestedVirtualization = true plus .param.NESTED_VIRT = "true" for the
template's in-guest /dev/kvm verification (vergil-project/vergil-vm#131).

The composed-spec fingerprint folds nested in only when true, so
toggling the knob flips NEEDS-REBUILD while every profile that never
set it keeps its pre-upgrade fingerprint.

A host-support preflight (macOS 15+, M3-or-later Apple silicon) aborts
create — and rebuild, before the destroy half — with a clear message
naming the requirement. Lima's own rejection is the backstop; the
in-guest check is the last line.
