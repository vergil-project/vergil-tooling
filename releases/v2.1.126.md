
# Release 2.1.126 (2026-07-11)

## Features

- **allow the agent to run 'issue transfer' for finite-epic relocation (#2271)**
Add "transfer" to the issue allowlist in vrg_gh.py so human and user
identities can relocate a finite epic's issues to another repository via
gh issue transfer. The audit identity stays read-only because `issue`
is absent from its allowlist; no denied-pair entry is added since
transfer is a non-destructive repository move.

The _print_help() output is derived from _ALLOWED, so it reflects the
new action automatically. Tests add the (issue, transfer) allowed pair
and a USER-agent test asserting the call is forwarded to gh.
