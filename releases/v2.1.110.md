
# Release 2.1.110 (2026-07-04)

## Bug fixes

- **export ~/.local/bin PATH for bootstrap_volume vrg-git calls (#2146)**
The bootstrap-volume stage ran 'vrg-git clone'/'vrg-git fetch' as bare commands over a non-login 'gcloud compute ssh --command=' shell, whose PATH is the system default and excludes ~/.local/bin where 'uv tool install' places vrg-git. So both the clone (fresh volume) and fetch (reattach) branches failed with exit 127 (command not found) on every cloud rebuild — a regression from the git->vrg-git switch (#1791/#1786/#1781), since raw /usr/bin/git was on the default PATH.

Route both calls through 'bash -c' with a $HOME/.local/bin PATH export, mirroring the established pattern every other in-guest tool call uses (vm_guest's _PLUGIN_PATH_EXPORT / _uv_tool_install). Add regression tests asserting both branches' composed guest command carries the export.
