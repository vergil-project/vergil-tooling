
# Release 2.1.153 (2026-07-17)

- **make copy_claude_config tolerate the read-only projected CLAUDE.md (#2437)**
Interaction between #2412 (lock) and pre-existing copy_claude_config: lock_projection chmod a-w's ~/.claude/CLAUDE.md at the end of a cloud session; the next session's copy_claude_config runs before project_memory and did a plain 'cat > ~/.claude/CLAUDE.md', failing with 'permission denied' and aborting every 2nd+ cloud session. Now chmod u+w each target (2>/dev/null) before the cat, mirroring project_memory's guard. No-op on Lima and on a fresh file. Host-side change only. Ref vergil-project/.github#156.
