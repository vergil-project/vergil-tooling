
# Release 2.1.63 (2026-06-23)

## Bug fixes

- **copy host Claude config onto cloud VMs so bypass is reachable (#1826)**
The Lima path seeds host ~/.claude config into the VM via
copy_claude_config (carrying the operator's permissions.defaultMode), but
the cloud path's _cs_link_claude only linked history dirs — so
off-platform VMs never received the bypassPermissions default and could
not reach bypass mode. Bring the cloud stage to parity: copy host config,
then link. Add a direct stage test and stub copy_claude_config in the
cloud-create test bundle.
