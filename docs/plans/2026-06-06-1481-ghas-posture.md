# Declared GHAS Posture Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the GHAS posture a declared `vergil.toml` setting (defaulting to
visibility inference) and thread it through the CI gates ruleset, security
settings, and the CI workflow generator, so repos without GHAS get gates their
CI can actually satisfy.

**Architecture:** A new optional `[project] ghas` key resolves through one
helper (`ghas_available()`) in `github_config.py`. `compute_desired_state()`
resolves the flag once and passes it to `desired_security_settings()` and
`desired_ci_gates_ruleset()`, which drop GHAS-only features when false.
`render_ci_workflow()` independently catches up with the
`ci-security.yml@v2.1.3` caller contract (`actions: read`), emits
`upload-sarif: false` for private repos, and bumps all emitted pins to v2.1.

**Tech Stack:** Python 3.12, dataclasses, pytest, unittest.mock

**Spec:** `docs/specs/2026-06-06-ghas-posture-design.md`
**Issue:** vergil-project/vergil-tooling#1481

All file paths are relative to the worktree root
(`.worktrees/issue-1481-ghas-posture/`). Run all commands from inside the
worktree. Commit with `vrg-commit` (raw `git`/`gh` are blocked; use
`vrg-git` for staging).

def ghas_available(config: VergilConfig, *, visibility: str) -> bool:
    """Resolve the GHAS posture for a repo.

    A declared ``[project].ghas`` in ``vergil.toml`` wins; otherwise
    GHAS is inferred as available exactly when the repo is not private.
    """
    if config.project.ghas is not None:
        return config.project.ghas
    return visibility != "private"

```python
def desired_ci_gates_ruleset(
    project: ProjectConfig,
    ci: CiConfig,
    *,
    ghas: bool,
) -> DesiredRuleset:
    """Derive the CI gates ruleset from project identity and CI config."""
    checks: list[dict[str, object]] = []
    lang = project.primary_language

    # Always present
    checks.append(_make_check("quality / common"))
    checks.append(_make_check("security / trivy"))
    checks.append(_make_check("security / semgrep"))
    checks.append(_make_check("security / standards"))

    # GHAS check runs — created by GitHub Advanced Security (app 57789)
    # when workflows upload SARIF via codeql-action/upload-sarif.  These
    # gate on whether the PR introduces new alerts in changed lines.
    # Without GHAS the check runs can never materialize, so requiring
    # them would block merges forever; the trivy/semgrep jobs still gate
    # on findings via scanner exit codes.
    if ghas:
        checks.append(_make_ghas_check("Trivy"))
        checks.append(_make_ghas_check("Semgrep OSS"))

    # CodeQL for supported languages — requires GHAS-backed code scanning
    if ghas and lang in _CODEQL_SUPPORTED_LANGUAGES:
        checks.append(_make_check("security / codeql"))
        checks.append(_make_ghas_check("CodeQL"))
```

(The rest of the function — versioned checks, integration tests,
