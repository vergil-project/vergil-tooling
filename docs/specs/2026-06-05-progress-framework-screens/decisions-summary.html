vrg-progress — design decisions
TopicDecision
Output rendering
Interactive (TTY) ✓Rolling last-N window via rich live display. --output-window N (default 5). 0 = full stream then collapse.
CI — GitHub Actions ✓::group:: / ::endgroup:: annotations. Each stage is a collapsible section.
CI — other / piped ✓Flat verbose: start marker → all output → completion marker.
Override ✓--output-format [rich|gha|plain] overrides auto-detect.
Library ✓rich added as first external dependency in vergil-tooling.
Failure model
Failure modes ✓warn — yellow ⚠, non-fatal, exit 0  ·  fail_defer — red ✗, run continues, exit 1 at end  ·  fail_fast — red ✗, immediate abort, exit 1
Escape hatches ✓Specific fail_fast stages may declare a skip_flag. Passing --skip-<stage> demotes that stage to warn for that run. Shown in summary.
Final summary ✓Always printed. Lists all warnings and deferred failures with one-line descriptions. Verbose output pointer: .vergil/<cmd>-YYYYMMDD-HHMMSS.log
Stage API
Declaration style ✓Declarative STAGES = [Stage(...), ...] list at top of command. Pipeline runner iterates it. No inline one-offs — if it's a stage, it's in the list.
Stage fields ✓name, fn, mode (warn/fail_defer/fail_fast), optional skip_flag
Scope
Which commands ✓Any procedural command that runs multiple steps over more than a few seconds. vrg-release is first. Atomic quick commands (vrg-commit etc.) excluded.
Module location ?Shared module in vergil_tooling/lib/progress.py — imported by any command that needs it. TBD: confirm name.
