Phase 5 — the same handshake, now on the PR

The bus is the PR (commits · check statuses · review threads), the wait is the user agent monitoring it, and the gate is now hard. New vs the local loop: two feedback sources run at once (mechanized CI + audit), and the record is public/archived.

Handoff (start of phase): opening the PR auto-triggers the mechanized CI gates — the human doesn't start them. The human then hands the PR URL to both: → user agent "monitor till green" · → audit agent "re-review on the PR."

Soft → hard. Pre-PR audit was advisory (withhold a file). Here the audit's "request changes" is a real merge block, and the record lives publicly on the PR.

Three ways it ends (mirrors the local loop)

① Green + approved

All checks pass and the audit review is approved → PR is mergeable → human merges (Phase 6).

② User agent gives up

Can't get CI green or can't satisfy the review → stops, asks the human. Same everyday wall as the local loop.

③ Audit blocks & holds

Requests changes that stay unresolved → merge stays blocked. Public, on the PR. Human decides what happens next.
