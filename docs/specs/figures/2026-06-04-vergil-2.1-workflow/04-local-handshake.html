implement ↔ audit — worked example + the three ways it ends

The sequence below is the warning→fix→approve path with a real example. Immediate-approve is the same minus the loop. The two other endings — agent gives up, or audit declares an error — are below.

Three ways the loop ends

① Approve

Immediately, or after warnings are fixed. Feedback file's INFO lists the signed-off commit IDs. User agent tells the human it's ready for the PR.

normal · expected

② User agent gives up

"I can't get this green / I don't know what to do." The user agent stops and asks the human — the everyday case where an agent hits a wall. Nothing exotic; you step in.

approval bit = punted

③ Audit declares an ERROR

Not a fixable warning. The audit agent does NOT write the feedback file — it stops and alerts the human. The user agent stays safely parked on its await (blocking-forever is the feature). This is a non-human-approved exception — e.g. a net-new security suppression. How approval gets recorded/annotated = deferred.
